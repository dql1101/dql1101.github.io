MySql数据库性能优化

发表于 2017-08-27 | 分类于 开发心得
字数统计: 962


MySql数据库

关系型数据库，将数据保存在不同的表中，而不是将所有数据放在一个大仓库内，这样就增加了速度并提高了灵活性。

###MySql的性能优化

  1. 内核优化；进行压力测试之后，在my.cnf中对配置参数进行优化；

  1. 开启查询缓存(mysql默认开启查询缓存)，但是在查询过程中，查询缓存机制对某些函数例如：NOW(), RAND(), CURDATE()等不起作用，所以需要用一个变量来代替这个函数从而开启缓存：

  2. EXPLAIN我们的查询语句。在select语句之前加上explain可以让我们看到sql语句潜在的性能问题，从而根据需求做出改进。

  3. 当只要一行数据时使用LIMIT 1；MySql数据库引擎会在找到一条数据后停止搜索，而不是继续往后查找吓一跳符合记录的数据。

  4. 为我们经常需要搜索的字段建立索引，而且在查询的时候不要使用双%的模糊查询，例如like ‘%Love%’；因为这样子会使我们的索引失效,而且我们在查询的时候不要使用select * 来做查询。

  5. 在Join表的时候使用相当类型的例，并建立索引。这样，MySQL内部会启动为你优化Join的SQL语句的机制。而且，被用来Join的字段，应该是相同的类型的。例如：如果你要把 DECIMAL 字段和一个 INT 字段Join在一起，MySQL就无法使用它们的索引。对于那些STRING类型，还需要有相同的字符集才行。(两个表的字符集有可能不一样)

  6. 想获得随机排序的数据，不要使用ORDER BY RAND()；可以将查询出来的数据放到set集合中来完成查询数据的乱序。

  7. 因为null值在存储的时候也是需要额外空间的，所以除非我们确实需要在表中使用null值，在其他情况下我们尽可能多的使用NOT NULL来约束字段；使用垂直分割的手段，将数据库中的表按列变成几张表，这样可以降低表的复杂度和字段的数目，从而达到优化的目的。

  1. 使用PreparedStatement作为预处理对象来处理sql语句，可以保证程序不会受到”SQL注入攻击”，增强性能和安全性。将表中字段的长度固定起来也可以提升性能，并且，固定长度的表也更容易被缓存和重建。不过，唯一的副作用是，固定长度的字段会浪费一些空间，因为定长的字段无论你用不用，他都是要分配那么多的空间。

  2. 选择正确的存储引擎；查询数据多的时候使用MyISAM引擎，增删改数据多的时候使用InnoDB引擎。
